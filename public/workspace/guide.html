Blenderアドオンの仕組みと正しい使い方

⚠️ まずこれを最初に確認してください（初心者の方へ）

本サービスは 「レンダーファーム」です。GeForce NOWやクラウドPCとは異なります。 ビューポートのリアルタイム描画（Renderedプレビューなど）は引き続き手元PCの底力で処理されます。

🖥️

「Solidモード」で作業する（強く推奨）

Blenderのビューポート左上のドロップダウンを「ソリッド」に。リアルタイムレンダリングはオフになり、手元PCの負荷が極小になります。

☁

「最終レンダリング」だけクラウドへ投げる

モデリング完了後、アドオンから「クラウドレンダリング開始」をクリック。.blendファイルがサーバーにアップロードされ、GPUが高品質レンダリングを実行します。

🔄

レンダリング中は次の作業を続けられる

クラウドでレンダリング中、手元のBlenderで次のシーンのモデリングを続けられます。完成したらアドオンの「DL」ボタンで結果をダウンロード。

📈

高負荷なアニメーションに最適

例: 1920x1080 / 500サンプル / 300フレーム → 手元PCで数時間がかかる処理も、RTX 4090で数分で完了。

💡 将来の機能: ストリーミング操作について

現在水面下で、ビューポート自体もクラウドで処理し、リモート操作できる「ストリーミング方式」を検討中です。GeForce NOWのようにビューポート描画がクラウドで実行される機能は、実験的に実装を検討しています。請うこうご期待ください。

1

ワークスペースとは？

ワークスペースは、ブラウザの中でGPUマシンを操作できる開発環境です。インストール不要・特別なソフトも不要。ブラウザを開くだけで、高性能GPUを搭載したPCのターミナルが使えます。

📺

ブラウザターミナル

コマンドを入力して、GPUマシン上でプログラムを実行できます。Python、CUDA、pip、git など使い慣れたツールがそのまま使えます。

📁

ファイルマネージャー

左サイドバーでファイルの一覧を確認。ファイルをクリックするだけでダウンロードでき、アップロードボタンでファイルを送信できます。

📊

GPUモニター

GPU使用率・温度・VRAM使用量をリアルタイムで確認できます。学習の進行状況を監視するのに便利です。

2

どんなときに使うの？

「GPUを使うとき」と言われてもピンとこない方へ。以下のような作業でGPU環境が活躍します。

🤖

AI・機械学習

PyTorchやTensorFlowでモデルの学習を実行。自分のPCでは何時間もかかる学習が、GPUなら数分で完了。

🎨

画像生成AI

Stable Diffusion、ComfyUI、FLUX等で高品質な画像を生成。VRAMが大きいほど高解像度の画像が作れます。

💻

AIコーディング

Claude Code（OpenClaw）などのAIコーディングツールをGPU環境で使用。大規模プロジェクトの高速処理に。

🎬

動画レンダリング

Blender、FFmpegなどでGPUアクセラレーションを利用した高速動画レンダリング。

🧬

科学計算

分子シミュレーション、流体力学、データ分析などCUDAを活用した大規模計算処理。

📊

LLMファインチューニング

ChatGPTのようなLLM（大規模言語モデル）を自分のデータで追加学習。カスタムAIを作成。

3

ワークスペースに入る方法

1

ポータルにログイン

janction.net/portal/ にアクセスして、アカウントでログインします。

2

GPUを予約

使いたいGPUを選んで予約します。ポイントで支払い、利用時間を選択します。

3

ワークスペースを開く

予約時間になると、ポータルに「ワークスペースを開く」ボタンが表示されます。クリックするとブラウザで開発環境が開きます。

4

開発スタート！

ターミナルが表示されたら準備完了。コマンドを入力してGPU環境で作業を始められます。

4

ターミナルの基本操作

ワークスペースのターミナルは、LinuxやMacの「ターミナル」と同じように使えます。

🔰 最初に試すコマンド

# GPUの状態を確認
nvidia-smi

# 現在のフォルダを確認
pwd

# ファイル一覧を表示
dir

# Pythonのバージョンを確認
python --version

📋 よく使うコマンド

やりたいことコマンド
GPUの状態を確認nvidia-smi
Pythonスクリプトを実行python train.py
pipでパッケージをインストールpip install torch numpy
ファイル一覧を見るdir
フォルダ移動cd フォルダ名
ファイルの中身を表示type ファイル名
Gitでコードを取得git clone https://github.com/...
実行中のプログラムを停止Ctrl + C
5

ファイルのアップロード

自分のPCにあるデータやコードをGPU環境にアップロードできます。

1

左サイドバーの「⬆」ボタンをクリック

ファイルマネージャーの上部にあるアップロードボタンを押します。

2

ファイルを選択

ファイル選択ダイアログが開くので、アップロードしたいファイルを選びます。

3

アップロード完了

プログレスバーが100%になったら完了。ファイルマネージャーに表示されます。

💡 ヒント: Gitを使う方法もあります

GitHubにコードがある場合は、ターミナルで git clone するのが最も簡単です。

6

成果物のダウンロード

学習済みモデル、生成した画像、レンダリング結果など、GPU環境で作成したファイルを自分のPCにダウンロードできます。

1

左サイドバーでファイルを探す

ファイルマネージャーでフォルダを移動して、ダウンロードしたいファイルを見つけます。

2

ファイル名をクリック

ダウンロードしたいファイルをクリックすると、自動的にダウンロードが始まります。

💡 ヒント: 大きなファイルの管理

学習済みモデル（数GB）など大きなファイルは、Google DriveやHugging Faceにターミナルから直接アップロードすることも可能です。

7

GPUを使った具体的な作業例

🤖 例1: PyTorchでAI学習を実行

# PyTorchをインストール（初回のみ）
pip install torch torchvision

# GPUが使えるか確認
python -c "import torch; print(torch.cuda.is_available())"
True

# 自分の学習スクリプトを実行
python train.py --epochs 100 --batch-size 64

🎨 例2: Stable Diffusionで画像生成

# リポジトリをクローン
git clone https://github.com/AUTOMATIC1111/stable-diffusion-webui.git
cd stable-diffusion-webui

# 起動
python launch.py --listen

💻 例3: Claude Code（AI開発アシスタント）

# Claude Code をインストール
npm install -g @anthropic-ai/claude-code

# APIキーを設定（自分のキーを使用）
set ANTHROPIC_API_KEY=sk-ant-xxxxx

# プロジェクトフォルダで起動
cd my-project
claude

🎬 例4: FFmpegでGPUエンコード

# GPUアクセラレーション（NVENC）でH.264エンコード
ffmpeg -i input.mp4 -vcodec h264_nvenc -preset p5 output.mp4

⚠️ 注意: セッション時間

予約した時間を過ぎるとセッションが自動終了します。長時間の学習タスクは、予約時間に余裕を持って計画してください。

便利なTips

ヒント説明
GPU使用率を確認するワークスペース右上のGPUモニターでリアルタイムに確認できます。使用率が0%なら、プログラムがGPUを使っていない可能性があります。
学習を中断・再開する学習途中のチェックポイントを保存するようにしましょう。torch.save(model, 'checkpoint.pth')
大量ファイルの管理ターミナルで tar -czf results.tar.gz output/ してからダウンロードすると効率的です。
GPUメモリ不足（OOM）バッチサイズを小さくするか、torch.cuda.empty_cache() でメモリを解放してみてください。
コマンドの履歴キーボードの ↑ ↓ キーで過去のコマンドを呼び出せます。
画面を広く使うブラウザをフルスクリーン（F11）にすると作業しやすくなります。
